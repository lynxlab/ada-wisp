Configurazione

  1. copiare tutti i file, e l'albero delle directory, nella directory di root

  2. creare almeno 2 DB (ogni provider ha il suo DB. Se ad es. hai 2 provider, hai bisogno di 3 DB, un DB comune, ed 2 DB per ) ciascun provider).

    • ada_common --> DB common.
    • ada_provider0 --> DB del provider 0.
    • ada_provider1 --> DB del provider 1.

  3. importare lo script ada_common.sql in ada_common

  4. importare lo script ada_provider0.sql in ada_provider0

  5. importare lo script ada_provider_empty.sql in ada_provider1 (se hai bisogno di un altro provider)

  6. rinominare il file config_path_DEFAULT.inc.php in config_path.inc.php

  7. modificare il file config_path.inc.php valorizzando correttamente la define ROOT_DIR, es. se hai copiato i files e le directory in /var/www/wisp:

    • define('ROOT_DIR','/var/www/html/wisp');

  8. rinominare la directory config/config_install_DEFAULT.inc.php in config/config_install.inc.php

  9. modificare la directory config/config_install.inc.php

    • valorizzare correttamente * WISP Common database
    • valorizzare correttamente * WISP default provider
    • valorizzare correttamente * Default admin mail address
    • valorizzare correttamente URL. Fare attenzione a non rimuovere the trailing: // js_import define('HTTP_ROOT_DIR','http://ada.lynxlab.com');
    • opzionalmente modificare il Default template family (solo se vuoi utilizzare un differente tema di layout)
    • opzionalmente modificare la define('PORTAL_NAME','ADA - e-learning ');

  10. rinominare la directory clients_DEFAULT in clients

  11. modicare la directory clients/client0/client_config.inc.php (ogni provider ha la sua propria directory. es.: provider 1 avrà client1 come directory di riferimento)

    • cambiare i parametri di accesso al DB
    • cambiare,in caso, il timezone del provider

  12. se hai più di un provider devi creare più di un file client_config.inc.php es.: 2 providers. Devi configuare il file clients/client0/client_config.inc.php e il file clients/client1/client_config.inc.php.
    Ogni file client_x/client_config.inc.php deve essere valorizzato in questo modo:
    es: client1/client_config.inc.php deve avere il nome delle costanti così: CLIENT1_DB_NAME, CLIENT1_DB_USER, CLIENT1_DB_PASS, CLIENT1_DB_HOST, CLIENT1_TIMEZONE

  13. cambiare i permessi della directory services/media/ in modo tale che il web server sia in grado di scriverci dentro

  14. cambiare i permessi della directory upload_file/ in modo tale che il web server sia in grado di scriverci dentro

  15. cambiare i permessi della directory docs/ n modo tale che il web server sia in grado di scriverci dentro

  16. cambiare le informazioni del file di news: browsing/news_language (ex.: news_en per le news in inglese) oppure accedere alla pagina edit_news.php dopo aver effettuato il login come amministratore

  17. configurare i widgets caricati in home page copiare il file widgets/main/index_DEFAULT.XML in widgets/main/index.xml accedere al link "configurare i widget" di questo maunale, per capire come configurare correttamente i widget.

Installare moduli

  • PREREQUISITI

  • Se l'installazione non supporta i moduli, si prega di aggiornare i seguenti file di installazione (prendendoli dall'ultima versione di ADA)

    • * include/Forms/lib
    • * include/layout_classes.inc.php
    • * include/output_classes.inc.php
    • * config/ada_config.inc.php
    • * config/config_install.inc.php

  • INSTALLAZIONE

  • Importare / Eseguire nel database common il file: ada_NOME_MODULO_module.sql
  • Copiare i file tutti i file contenuti in * modules/NOME_MODULO/*
  • Assicurarsi di creare una cartella layout per ogni layout esistente (o usato) nell'installazione

  • CONFIGURAZIONE

  • Aggiungere (o creare) nel file config/config_modules.inc.php

    • // es.: defines for modules test
define('MODULES_TEST_PATH', ROOT_DIR.'/modules/test');
if (file_exists(MODULES_TEST_PATH.'/index.php')
 && file_exists(MODULES_TEST_PATH.'/edit_test.php')
 && file_exists(MODULES_TEST_PATH.'/tutor.php')) {

    require_once(MODULES_TEST_PATH.'/config/config.inc.php');

    define('MODULES_TEST', true);
    define('MODULES_TEST_HTTP', HTTP_ROOT_DIR.'/modules/test');
} else {
    define('MODULES_TEST', false);
}

Configurare widget

  • Rinominare il file widget/main/index_DEFAULT.xml in widget/main/index.xml: questo sarà il file di configurazione per i widget che compariranno nella pagina di index,qualsiasi altro widget da inserire in un' altra pagina deve seguire il path del file.tpl nel quale desideriamo che compaia il widget. Quindi a titolo esplicativo:

    • Se il template si trova in: layout/TEMA/templates/XXX/YYY.tpl
    • il file di configurazione del widget dovrà essere posizionato in: widget/XXX/YYY.xml
    • Se l'ambiente è NON-MULTIPROVIDER, ed esiste il seguente file :clients/PROVIDER_NAME/widgets/XXX/YYY.xml, questo avra priorità su qualsiasi altro.

    qualsiasi tag <widget> deve contenere i seguenti tag:

    • * <field> che valorizzerà il nome del template_field contenuto nel file.tpl dove apparirà il widget.1
    • * <id> questo valorizzerà l'id che sarà assegnato al div html che conterrà il widget.2
    • * <module> questo è il nome del file contenuto nella directory widgets/ajax che sarà eseguito per permettere la visualizzazione del widget.
    opzioni di rendering:

    • * <active> è un parametro opzionale 0/1 che stabilisce se il widget è attivo o no.
    • * <async> è un parametro opzionale 0/1 che stabilisce se il widget è caricato in modo asincrono (1) o in modo sincrono (0)
    • * <param> usato per passare dei parametri al file indicato in <module>.

    1. Richiesto per il widget che lavora sia in modalità sincrona che asincrona.
    2. Richiesto per il widget che lavora in modalità asincrona.

    Note:
    • Se si desidera utilizzare il modulo Twitter, bisogna configurare un access token:

      • * rinominare il file TwitterAccess_DEFAULT.inc.php in TwitterAccess.inc.php ed inserirlo nella directory: widgets/ajax/include
      • * configuare il proprio accedd token.

Configurazioni extra

  • Welcome Message Mail

    • E' possibile cambiare il messaggio di benvenuto spedito a un'utente che si è appena registrato: accedere al seguente file e modificarlo: /docs/welcome_language.txt (ex.: /docs/welcome_en.txt english message).

    Help

    • La directory docs contiene un file di help per l'utente. Accedere al file e modificarlo: /docs/help_language.txt (ex.: /docs/help_en.txt english message).
    • E' possibile inoltre cambiare il file di help seguendo questa procedura alternativa:
      • * Eseguire il login come amministratore.
      • * Accedere alla voce di menù "Agisci".
      • * Cliccare su "Edit help content".
      • * Cliccare sul file che si desidera modificare.
      • * Eseguire le modifche desiderate. E' possibile inserire HTML tags.

    News

    • La directory docs contiene un file di news per l'utente. Per modificarlo seguire la procedura sopra descritta per il file di help.

    Info

    • La directory docs contiene un file di info per l'utente. Per modificarlo seguire la procedura sopra descritta per il file di help.

    Corso pubblico

    • E' possibile settare un corso pubblico:

      • In un ambiente MULTIPROVIDER, il corso dovrà essere creato nel provider0, settare la costante PUBLIC_COURSE_ID_FOR_NEWS definita in config/config_install.inc.php con l'id del corso destinato ad essere un corso pubblico.
      • In un ambiente NON-MULTIPROVIDER ogni providerX può avere il proprio corso pubblico: settare la costante PUBLIC_COURSE_ID_FOR_NEWS definita in clientX.inc.php con l'id del corso destinato ad essere un corso pubblico.
