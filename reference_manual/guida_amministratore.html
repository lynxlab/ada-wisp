Struttura della piattaforma

  • In Holis, ogni provider eroga un certo numero di servizi.I servizi sono salvati nel DB il cui nome è definito nel file clientX/client_conf.inc.php.
  • Ogni servizio ha una specifica tipologia. Le tipologie di servizio sono definite nel file config/config_main.inc.php.
  • Ogni servizio creato dal provider è automaticamente linkato nella piattaforma comune (salvato,cioè, nel DB common).
  • Ad ogni servizio viene automaticamente associata un'instanza per permettere agli studenti di iscriversi ad essa.
    Come fare per creare un nuovo servizio:

    • * eseguire il login come switcher.
    • * cliccare sulla voce di menu "agisci".
    • * cliccare sulla voce di menu "aggiungi servizio".

Provider

  • L'amministratore è l'unico account che consente l'aggiunta,la rimozione o la modifica di un Provider.
    Come aggiungere un provider:

    • creare un nuovo DB.
    • importare lo script sql ada_provider_empty.sql nel nuovo db.
    • modicare la directory clients/client0/client_config.inc.php (ogni provider ha la sua propria directory. es.: provider 1 avrà client1 come directory di riferimento)

      • cambiare i parametri di accesso al DB.
      • cambiare,in caso, il timezone del provide.r

    • Il file client_x/client_config.inc.php deve essere valorizzato in questo modo:
      es: client1/client_config.inc.php deve avere il nome delle costanti così: CLIENT1_DB_NAME, CLIENT1_DB_USER, CLIENT1_DB_PASS, CLIENT1_DB_HOST, CLIENT1_TIMEZONE.
    • Eseguire il login come amministratore.
    • Accedere alla voce di menù "Aggiungi provider".
    • Riempire corretamente i campi della form facendo attenzione a valorizzare il campo "Puntatore al database" con lo stesso nome della directory contenuta nella directory clients (ad es. clients1)
    • Creare un coordinatore (switcher) per il provider appena

Utenti

  • In Holis i primi utenti sono definiti nel provider default. Questi sono:

    • * adminAda --> admin.
    • * autoreHolis --> utente di tipo autore.
    • * tutorHolis --> utente di tipo tutor.
    • * switcherHolis --> utente di tipo switcher.
    • * studenteHolis --> utente di tipo studente.

    Le relative password sono le stesse degli username.
